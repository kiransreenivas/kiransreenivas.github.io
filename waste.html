The Problem

India is the world's second-most populous country, producing approximately 62 million tonnes of municipal solid waste (MSW) per year. Of this, only 70% is collected and just 20% gets properly treated — the remainder is dumped in open areas, river sites, and uncontrolled landfills, contaminating land, water, and threatening public health.

A core reason for this failure is that waste is not segregated at the source. Mixed waste is far harder and more expensive to process than pre-sorted streams. Metallic waste that could be recycled ends up in landfills. Wet organic waste that could be composted contaminates recyclable materials. The entire downstream system becomes less efficient and more costly.

Current industrial-scale segregation plants exist, but they are expensive, centralised, and require the waste to be transported before being sorted. The goal of this project was to design a compact, low-cost, automated system that segregates waste at the source — eliminating the need for manual sorting and reducing the hazards faced by waste workers.

The Physical Prototype
Built and Demonstrated — A Real Working Machine
The actual fabricated prototype — conveyor belt system with Arduino UNO controller (visible in blue), sensors mounted along the belt, and collection bins below

This was not a simulation or software project — it is a physically fabricated machine built and demonstrated at the final project review at RNS Institute of Technology, Bengaluru.

The team designed the mechanical frame, sourced components, assembled the conveyor belt system, wired the sensor array, and programmed the Arduino UNO microcontroller in Embedded C — all from scratch.

The machine successfully demonstrated the automatic segregation of metallic and dry waste in a live demonstration, with the Arduino controller making real-time decisions based on sensor inputs.

The blue glow visible in the prototype photo is the Arduino and electronic control system operating — a working embedded system controlling physical hardware.

System Design
How the Segregator Works
1
Waste Input via Hopper
Mixed waste is deposited into the hopper, which feeds it onto the conveyor belt. The system is designed to handle various waste types simultaneously.
2
IR Sensor — Presence Detection
An infrared (IR) proximity sensor detects the presence of any object on the belt and activates the conveyor motor, starting the segregation process automatically.
3
Capacitive Sensor — Dry Waste
The first active sensor uses capacitance to identify dry materials (plastic, paper, wood). When detected, a DC motor deflects the item into Bin 1 (dry waste).
4
Moisture Sensor — Wet Waste
Remaining items pass through a moisture sensor that detects organic or wet materials. Positive detection deflects items into Bin 2 (wet waste).
5
Inductive Sensor — Metallic Waste
The final inductive proximity sensor identifies metallic objects using electromagnetic induction and routes them into Bin 3 (metallic waste) for recycling.
6
Bin-Full Monitoring
The system monitors bin capacity and can alert authorities when bins are full — reducing unnecessary collection runs and optimising waste management logistics.

Technical Specifications
Microcontroller
Arduino UNO — all components programmed using Arduino IDE in Embedded C. The controller reads sensor inputs and activates output motors based on programmed logic.
Sensors Used
IR proximity sensor (presence detection), capacitive proximity sensor (dry waste), moisture sensor (wet waste), inductive proximity sensor (metallic waste)
Mechanical Components
Custom fabricated frame body, conveyor belt, rollers, DC motors for belt movement and waste deflection, collecting tanks for each waste stream
Programming
Embedded C via Arduino IDE — sensor logic programmed as a sequential decision tree: IR → Capacitive → Moisture → Inductive. Each sensor triggers a specific DC motor output.
Applications & Impact

🚗
